# birdbench 设计白皮书：一个可信的多模型鸟类识别评测台

> 本文讲**为什么这样设计**（方法论 + 取舍 + 失败模式）。可执行规格见 `docs/DESIGN.md`；领域词汇见 `CONTEXT.md`。

## 0. 它测什么，为谁
一张鸟图 → 让多家多模态 API 开集识别 → 判它说对没有 → 出**跨模型排行榜（准确率 + 成本）**。两个面孔共用一个内核：**(a)** 可复现的基准；**(b)** 产品同事的选型工具（"哪个 API 又准又省"）。

---

## 1. 核心难题：自由文本 → 对错
模型看图后吐**自由文本**：`Northern Cardinal` / `a red cardinal` / `Cardinalis cardinalis` / `北美红雀` / `某种雀`。评测要一个函数 `score(模型输出, 真值) → 对/错`。**全部难点在于同一个种有无数说法**——名字是脏的（大小写、别名、学名、语言、改名、拆分、修饰词）。

---

## 2. code-to-code：锚定到唯一且稳定的 ID
不比名字，比 **eBird `speciesCode`**（如 `norcar`）——唯一、跨语言、且因**pin 分类库版本**而稳定的短码。

```
真值侧(建集时定一次): "Northern Cardinal" ──解析器──▶ norcar   （存成码，可人审）
预测侧(每次评测):     模型说 "北美红雀"    ──解析器──▶ norcar
打分:                 norcar == norcar → 对              （纯码相等）
```

- **gold 怎么来**：策展用可靠俗名选种 → 解析器映射出码 → 拉 iNat research-grade（社区已核验是该种）图 → 存 `(图, gold=norcar)`。真值是**码**，一次定好、可复核（`verification` 字段记来源，负责人抽检）。
- **打分 = 两个码相等吗**。客观、确定、可复现、跨语言、评分时**零模型参与**。
- **pin 版本是地基**：gold 与预测锚在同一快照，码↔种永不漂移 → `norcar==norcar` 是稳定可复现的事实。用活的分类库，一次改名/拆分就让两侧码对不上。

---

## 3. 为什么不用字符串比对 / LLM 判等
| 做法 | 为什么否 |
|---|---|
| **字符串比对** | `"Northern Cardinal"` 对，但学名/中文/别名/大小写全判错；且 gold 存哪个字符串无解。太脆。 |
| **LLM 判等** | ① 不可复现（同输入不同判决）；② **自我偏袒虚高 10–25%**（LLM 给 LLM 打分）；③ 不可审计（"判官说的"没法查）；④ 可被话术钻空子；⑤ 慢且花钱。 |
| **code==code** | 确定、可复现、跨语言、可审计、无偏、瞬时免费。 |

---

## 4. 解析器：确定性阶梯子（脏活的归宿）
"名字→码"的映射由一条**确定性梯子**做，**先命中先赢**、**纯代码无 LLM**（同输入永远同输出）：

`NORMALIZE → EXACT_CODE → EXACT_SCI → EXACT_COM → 中文别名 → 同义(gazetteer) → 剥修饰(括号/morph/albino) → 亚种上卷(三名→二名) → 模糊学名(JaroWinkler+严格 margin/edit) → 缩写码(4字母,唯一才认) → LLM_NORMALIZE(V1-8,可选) → ABSTAIN`

**两条铁律**：
- **拿不准就 ABSTAIN**，绝不瞎猜。多义（一个名映射到多个种）→ 直接弃答。
- **每阶可审计、可单测**（`Cooper's Hawk→coohaw` 能写死）。模糊阶有 margin+编辑距离守卫，防误判。

---

## 5. 为什么不让模型直接输出 eBird 码
- 码是**不透明数据库约定**（`norcar`/`y00478`），模型在**名字**上训练、不可靠地知道码 → 逼它输码只会瞎编。
- 逼输码会把"认不认得种"和"背不背得码表"混在一起——我们只想量前者。
- **职责分离**：模型做擅长的（自然语言命名），确定性代码做记账（名→码）。这是 12-factor 的体现：LLM 产自然输出，确定性代码结构化。

---

## 6. 打分：四桶分离 + 两准确率
每个 `图×模型` 落一个桶（`score.py`）：

| 桶 | 含义 | 记谁的账 |
|---|---|---|
| **A** | top-1 解析码 == gold | 对 ✓ |
| **B** | 解析到合法码 ≠ gold（认错种）| **模型**的账（记 mistake 高度）|
| **C1** | 给了名却没解析出码 | **解析器**覆盖漏洞 |
| **C2** | 模型幻觉/非法输出（V1 影子）| 模型的账 |
| **D** | 模型弃答 | 模型选择弃权 |

**两个准确率**——隔离"模型能力" vs "整条管线"：
- **端到端** `end_to_end = A / 非弃答`：整条管线的实际表现。
- **解析器条件** `resolver_conditional = A / (A+B)`：**给定解析出了种**，模型判对的比例 → 剔除解析器覆盖影响，纯看**模型的种区分力**。

> 四桶的意义：把 **C1（解析器欠账）** 和 **B（模型认错）** 分开 → 解析器 bug **不会伪装成模型错误**。这是防止"用解析器缺陷冤枉模型"的关键，也让我们知道该修解析器还是该怪模型。

---

## 7. 分层部分分（不是非对即错）
- **top-k**：top-1/3/5 命中（模型给候选列表）。
- **目/科/属/种逐级**：认错种也看错到哪一层（同属 vs 跨目）。
- **LCA 距离部分分**：`1 − 距离/4` → 同种 1、同属 .75、同科 .5、同目 .25、跨目 0。
- **Mistake Severity**：认错时错得多离谱（把 Cooper's 认成 Sharp-shinned 情有可原；认成鸭子就离谱）。

---

## 8. 解析器错了怎么办
- **漏一个对的名（假 ABSTAIN=C1）**：少给模型分 → 靠四桶把 C1 单列出来度量+修（synonym overlay #24）。
- **映射到错码（假阳性）**：最危险 → 梯子只在高置信命中给码、多义即弃答、每阶可审计单测。
- **决定性优势**：解析器错误是**系统性的**（对所有模型一视同仁）→ **不偏袒模型间的相对排名**。就算有漏，排行榜对比仍公平。这是 LLM 判官给不了的。

---

## 9. LLM 只当 extractor，永不当 judge（V1-8）
唯一允许 LLM 进解析路径的地方：**把凌乱文本擦成干净种名**（`"Northern Cardinal (yellow morph)"→"Northern Cardinal"`），擦完**仍走确定性解析+码相等打分**。**对错判定永不经 LLM**。护栏：约束 prompt（提取非猜测）、temp 0 + 缓存（可复现）、确定性 exact 先行只对尾巴调 LLM。真机验证：认错种 0/9 被翻成 gold（不虚高），合成正控 3/3 救回（机制有效）。

---

## 10. 统计诚实（别被"排行榜幻觉"骗）
小 n 下准确率差异多半不显著。所以：
- **McNemar 精确检验**（配对，b+c<25 用二项精确非卡方）+ **Holm 校正**（多重比较）。
- **Clopper–Pearson 置信区间**（精确、不靠正态近似）。
- **显著性聚类**：把"与榜首无显著差异"的模型标为并列 → 不吹嘘假第一。
- 报告**明说** n 多大、差异显不显著。成本数是硬的，准确率排名在小 n 下是方向性的。

---

## 11. 校准与自洽
- **校准（V1-3）**：模型多自信 vs 实际多对。用 AUROC_f / Brier / 过度自信差 / AURC（避开小 n 下不稳的固定分箱 ECE）。发现：各模型普遍**过度自信**。
- **自洽（V1-4）**：同图采样 N 次投票，**投票率 = 比口头置信更可靠的置信信号**；语义熵（"同义"=码相等）度量分歧。

---

## 12. 成本与性价比（V1-6）
- 价灌进 litellm（`configs/*price_overlay.json`），图像走 token 计费。
- 核心指标 **$/正确 ID**（成本 ÷ 答对数）——同事选型看这个。发现：doubao-lite-nothink 性价比完胜；**thinking 对细粒度识别纯浪费**（同分 2.3× 价）。
- 价为**挂牌价、待真实账单校准**（汇率/单价改一处即可）。

---

## 13. 评测集设计（V1-7）
- **111 图 / 37 种**，混合构成（~37 种 × 3 图，不同 iNat 观察=不同个体/角度/羽色）。
- **分层**：稀有度 × 易混度 × 目；9 个易混组（accipiter/gull/scaup/yellowlegs…）压易混路径。
- **dHash 去重**、iNat CC research-grade、逐图署名。
- **韧性**：fetch 带 503 退避重试 + 暂存-达阈值才替换（防"预清空+失败=删数据"）。

---

## 14. 评测维度
一个 `cell` = 维度的内容哈希（图 sha × 模型 × 参数 × prompt 哈希）= 缓存键。可扫的维度：
- **模型**（多家 API）、**thinking on/off**、**温度**（骑在 `ModelSpec.params`）。
- **prompt 版本**（`prompts/*.md`，同事可编辑、版本管理）——发现：**俗名优先**（纯学名掉 40–50pp）、**CoT 伤细粒度识别**。

---

## 15. 已知边界（v2）
- **污染**：iNat 公开图或在模型训练集内 → 绝对准确率或偏高，**相对**排名仍有效。彻底 post-cutoff 时间留出待 v2。
- **真值复核**：`inat_research_grade + owner_review_pending`，负责人抽检待办。
- **answerable-only**：research-grade 皆可答；"不可答"样本待 v2。
- **拆分/多义老名**：如 `Northern Goshawk`（快照已拆）→ 正确弃答，不硬塞（塞了=瞎猜=虚高）。

---

## 一句话本质
**用唯一稳定的码当锚，把"无数说法→一个种"的脏活交给可审计的确定性解析器，打分只做码相等——客观、可复现、无 LLM 偏袒；再用四桶分离、分层部分分、统计显著性把"谁更好"讲诚实。**
